Caracteristicas, funcionamiento e instalación de Sedna

Sedna, es un sistema de gestión de base de datos XML nativa de código abierto, implementado a partir de cero con todas las funciones de administración de una base de datos, para almacenar grandes cantidades de datos XML.
La aportación especial de la base de datos Sedna, es su estrategia de almacenamiento mejorado basado en esquemas (schema-driven storage strategy), que permite gran eficiencia a la hora de procesar las consultas y almacenamientos en la BBDD. Y un sistema de manejo de memoria muy novedoso.
En el apartado, dentro del almacenamiento, debemos mencionar también un elemento novedoso de Sedna, las relaciones de inclusión entre elementos: Se utilizan punteros directos, para representar relaciones de un nodo XML como padre, hijo con sus respectivos hermanos.Este método a diferencia de enfoques relacionales que requieren realizar “joins” para atravesar un documento XML, permite realizar esto, a través de un puntero directo.



Para instalar podemos seguir los siguientes pasos:

Como podemos ver desde su web, podemos descargar dicha base de datos, para múltiples plataformas, nosotros hemos escogido Linux porque al no tener interfaz gráfica, la consola de Linux nos parece mucho más amigable.

En la sección de documentación de su web, están todos los pasos más detallados de la instalación http://sedna.org/documentation.html por ello, no entraremos en mucho detalle.

Una vez, descargado el archivo .sh e instalado en el directorio deseado, por ejemplo /usr/var/, deberemos cambiar la configuración del kernel, para que permita tantos semáforos como Sedna necesita para su ejecución añadiendo la linea kernel.sem = 250 64000 32 256 al archivo /etc/sysctl.conf

Una vez hecho esto, podemos iniciar Sedna mediante el comando /bin/se_gov y ejecutar cualquier comando de los presentes en dicho directorio bin. (Desde la documentación de Sedna, podemos consultar que hace cada uno).

Uno de los más básicos es ./se_cbd mediante el cual creamos una base de datos, con la cual empezar a trabajar.

Además es recomendable acceder a la carpeta /examples/comandlines/ donde podemos ejecutar algún código de ejemplo para ver su funcionamiento.
